# Тестовое окружение Telegram (тестовые DC)

Прокси умеет направлять трафик к **тестовым дата-центрам** Telegram
(тестовое окружение) вместо продовских. Нужно разработчикам, которые тестируют
ботов/клиентов в тестовой среде Telegram, находясь за DPI-блокировкой.

## Зачем

Бот, созданный в тестовом окружении (через BotFather на тестовом DC, либо Bot API
`.../bot<token>/test/...`), доступен только аккаунтам из того же тестового окружения.
Чтобы подключиться к нему клиентом с ПК из сети, где Telegram блокируется,
удобно гонять трафик через локальный MTProto-прокси. Раньше прокси знал только
продовые маршруты: клиент в тест-режиме уходил на продовый DC и не проходил
авторизацию.

## Как это работает

Тестовые DC доступны через [тот же вебсокет-фронт](https://core.telegram.org/mtproto/transports#uri-format),
что и продовые — отличается только путь: `/apiws_test` вместо `/apiws`.

Telegram Desktop помечает тестовые DC сдвигом номера +10000 (10001–10003),
поэтому прокси распознаёт их сам — включать ничего не нужно. Номер DC
нормализуется, и тестовый трафик идёт обычным маршрутом с тестовым путём;
для прямого TCP используются IP тестовых DC (`1: 149.154.175.10`,
`2: 149.154.167.40`, `3: 149.154.175.117`).

Можно быть авторизованным в основном и тестовом аккаунте одновременно:
продовый трафик идёт как раньше, тестовый — в тестовое окружение.

## Принудительный тестовый режим (`force_test_dc`)

Клиенты без сдвига +10000 (например, Telethon/TDLib через MTProto-прокси)
сообщают тестовые DC как обычные 1-3 — прокси не может отличить их от
продовых. Для них есть принудительный режим: **весь** трафик уходит на
тестовые DC, продовые аккаунты в данном режиме работать перестанут.

**GUI:** Настройки → «Датацентры Telegram (DC → IP)» → чекбокс
**«Принудительно тестовое окружение (тестовые DC)»**.

**CLI:**

```bash
tg-ws-proxy --force-test-dc      # + ваши --secret / --port
```

## Ограничения

Принудительный режим глобальный: с включённой галочкой продовые аккаунты
через этот прокси не работают. Для параллельной работы клиента запустите
второй экземпляр на другом порту с `--force-test-dc`.
